STUDENT SIDE
Secure Assessment Client (SAC)
(Standalone Monitoring Application – Runs alongside LMS)

AUTHENTICATION, PROFILE & ROOM ENROLLMENT

• Student registration and login
• Account & Profile Management

Update full name, email address, and password
Minimum password length: 6 characters
Requires current password validation

• Custom Avatars

Upload (.png, .jpg, .jpeg)
Maximum file size: 5MB

• Displays available and assigned monitoring rooms as clickable room orbs
• Clicking an orb displays room details (subject name, schedule, status)

• Students may be enrolled in rooms through:

Manual instructor assignment
Valid 6-character alphanumeric room enrollment code

• Students must explicitly click and join selected rooms
• Join timestamp recording
• Room status visibility: Pending / Countdown / Active / Ended
• Real-time room notifications (starting soon, started, ended)

Notes:
• Public room browsing is not allowed
• Room enrollment is controlled via assignment or valid code
• Rooms act as permanent subject-based containers allowing multiple reusable monitoring sessions

CONTROLLED ROOM ACTIVATION

• Instructor initiates room start
• System triggers countdown phase (e.g., 10 seconds)
• Monitoring modules remain paused during countdown

After countdown:
• Monitoring automatically activates
• Visual indicator displays "SESSION STARTED"
• Monitoring status indicator remains visible throughout room

SECURE SESSION CONTROL WINDOW (SOFT LOCK INTERFACE)

A persistent auxiliary control window automatically launched by SAC during an active monitoring session.

Purpose:
• Maintains session awareness
• Prevents premature session exit
• Provides controlled completion workflow

Behavior:
• Automatically appears when session becomes Active
• Always visible on screen
• Cannot be manually closed, minimized, or terminated
• Operates independently from main SAC interface

Interface Elements:

Done Button
• Student presses when finished
• Sends Session Completion Request to server
• Notifies instructor
• Does NOT allow immediate exit

Leave Button
• Initially disabled
• Enabled only after instructor approval or room unlock
• Allows safe exit

Soft Lock Mechanism:
• Prevents unauthorized exit
• Ensures monitoring remains active until instructor release
• Eliminates need for Alt+Tab back to SAC
• Guarantees visible session state awareness

Instructor Interaction:
• Instructor receives completion signal via IMC
• Instructor unlocks student exit
• Server broadcasts unlock event to SAC

System Actions:
• SAC listens for LeaveUnlock signal
• Leave button activates
• Monitoring session terminates safely

Restrictions:
• Window cannot be closed via Alt+F4, taskbar close, or standard controls
• Closing attempts may be logged as behavior events (optional)

DECISION & LOCAL REPORTING

Behavioral Rule-Based Decision Engine (BRBDE)

Severity Scoring:
S1 (Minor): 10 points
S2 (Moderate): 20 points
S3 (Severe): 50 points

Risk Classification:
Safe: Total < 20
Suspicious: 20 – 49
Cheating: ≥ 50

Additional Functions:

DETECTION CONFIGURATION FEATURE

IMC – Detection Settings
• Configurable per room
• Editable only when room is Pending
• Locked once Active

Configurable options:
• Enable/Disable Clipboard
• Enable/Disable Process Detection
• Enable/Disable Focus Detection
• Enable/Disable Virtualization Check
• Enable/Disable Idle Detection
• Configure Idle Threshold (seconds)

SAC – Behavior Adaptation
• Requests room detection settings upon join
• Activates only enabled modules
• Disabled modules remain inactive

PSS – Detection Control Management
• Stores settings in RoomDetectionSettings table
• Only instructors may modify settings
• Prevents modification when Active
• Provides settings to SAC upon request
